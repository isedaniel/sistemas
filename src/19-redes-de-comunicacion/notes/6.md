# Capítulo 7. Protocolo de Ruteo BGP

Es un protocolo de enrutamiento. Para intercambiar información de enrutamiento
entre diferentes redes.

Introduce el concepto de _sistema autónomo_ (AS, _autonomous system_).

Identifica de forma única un conjunto de redes.

Tiene actualizaciones incrementales, por lo tanto hay distintas versiones.

Es un _protocolo político_: se levanta entre una empresa y otra. Se pueden
poner políticas de restricciones. Además se puede cobrar si superamos las
métricas establecidas por esas políticas.

Es un protocolo de vector de trayectoria: define una ruta como correlación
entre destino y atributos de la trayectoria.

Cada nube representa un sistema autónomo. Los routers hacen frontera contra los
sistemas autónomos. Los enlaces se muestran como aristas orientadas (flechas).

# Definiciones

_Tránsito._ Carga de tráfico. Si tenemos un exceso se puede poner un cargo
extra. Depende de las políticas del protocolo.

_Interconexión._ Intercambio de información de enrutamiento y de tráfico.

_Ruta default._ Cuando no hay ruta específica para el tráfico, tenemos una ruta
_default._

_Zona libre de rutas por defecto._ 

# Sistemas autónomos

Colección de redes bajo una misma política y protocolo.

Mismo propietario y control administrativo.

Identificado como un entero de 32 bits: `AS100`

Se suele abreviar AS.

# Número de AS (ASN)

Dependiendo de los números, tenemos una descripción.

Ejemplo: 1 - 23455 son ASNs públicos. De 64496 - 64511 es para uso en
documentación y código de ejemplo.

Con este número podemos identificar desde dónde viene el tráfico.

Estos números son asignados por los _Registros Regionales de Internet_ (RIRs).

Se pueden consultar en `PeeringDB`. Organización. Sitio Web. ASN. Prefijos de
IPv4 e IPv6. Tasa de tráfico. Entre otros datos.
