# Autómatas de Pila

Trabaja con símbolos, gramáticas, palabras. Pueden validar palabras.

Es una máquina de datos. Trabaja los símbolos de cierta manera. Les da un
orden. Un orden de llegada y de salida.

# Concepto de pila

La pila tiene una entrada que funciona también como salida.

Funciona como `LIFO: Last In First Out`. El último que entra es el primero que
sale.

Ingresamos 1, 2 y 3. 1 está al fondo y es el último en salir. 3 está arriba y
es el primero en salir.

Ese orden es lo fundamental para el concepto de pila y para el autómata de
pila.

# Llamados a funciones

Tengo una función `A()`. Esa función llama a `B()`, que llama a `H()`.

Para llevar el orden de esos llamados usamos el concepto de pila. Internamente
funciona como una pila.

(La famosa `Call Stack`).

Cuando `H()` termina, vuelve a la línea donde había quedado en `B()`. Termina
`B()`, vuelve a donde había quedado en `A()`. Todas esas posiciones se guardan
en la `Call Stack`.

# Sistema operativo

Tengo muchos programas. Cada uno tiene muchas funciones corriendo. Hay 4
núcleos ejecutando muchos procesos. Cómo hace.

Cada vez que la CPU se mueve entre procesos, el contexto para cada uno de los
procesos entra y sale de una pila (es más complejo, pero es la idea general).

# Autómata de pila

Utiliza este concepto. Con una estructura formal. Y se puede representar con
nodos y aristas.

# Grafo de pila

Similitudes con los grafos anteriores. Tiene nodos. Tiene aristas. Tiene
condiciones para que sigamos las aristas. Todo eso es similar.

La diferencia está en las reglas, que son los requerimientos para ir de un nodo
a otro.

Una regla tiene la forma: `a, Z; XZ`.

Tenemos `símbolos de entrada`. Como `a`.

El segundo elemento es el elemento arriba de la pila. `Z` estaría arriba de la
pila para que se cumpla esa regla.

El tercer elemento es lo que grabo en la pila. `XZ`.

# Autómata de pila

Permite validar una palabra del tipo 2. Por ejemplo: `L = {0^n1^n}`. Este tipo
de gramática la puede validar.

Permite validar otro tipo de palabras que los Autómatas Finitos Determinados no
podían.

Con reglas se puede definir. Pero con un `AFD` no. Con el autómata de pila
puedo contar elementos en la pila. Y asegurar que agregue tantos `1` como `0`,
cumpliendo con `L = {0^n1^n}`.

Básicamente le agregamos una memoria, que funciona como una pila.

# Definición formal del autómata de pila

Es una _séptupla_ (conjunto ordenado de 7 elementos):
$A_P = (Q, \Sigma, \Gamma, \delta, q0, Z0, F$

Siendo:

- $Q$      el conjunto finito de estados
- $\Sigma$ el alfabeto de entradas
- $\Gamma$ el alfabeto de pila
- $q0$     el estado inicial
- $Z0$     el símbolo inicial de la pila
- $F$      conjunto de estados finales
- $\delta$ función de transición:
$Q \times \{\Sigma \cup {\lambda}\} \times \Gamma \to Q \times \Gamma*$

(Combinaciones de estado, entrada y estado de la pila, con eso se a dónde voy y
cuál es el estado de la pila).

Alfabeto de entradas. Los símbolos que pueden ingresar.

# Configuración

Es la situación en uns instante. También _descripción instantánea_.

# Ejemplo

Autómata que de como válido el lenguaje `L = {a^nb^n / n >= 0}`.

Definimos el autómata.
Los estados posibles son `Q = {0, 1, 2}`. 3 nodos.

Las entradas `{a, b}`.
Símbolos de Pila `{Z, a}`.
Arranca en nodo `1`.

```c
Z -> q0 -> (a, Z, aZ) -> q1 -> (a, a, aa) -> q1 -> (b, a, \lambda) -> q2
q2 -> (b, a, \lambda) -> q2 -> (\lambda, Z, \lambda) -> q0
```

Este autómata asegura que tengamos tantas `a` en la pila como sean necesarias,
para que se incorporen todas las `b` necesarias para cumplir con el lenguaje.

No podemos llegar hasta el nodo final `q0` hasta que no igualemos la cantidad
de `a`. Solo validamos palabras que cumplan con `L = {a^n b^n / n >= 0}`.
