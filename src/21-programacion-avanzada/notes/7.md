# ORM

Mapeadores objeto-relacional (Object-Relational Mapping). Permiten traducir
conceptos orientados a objetos con conceptos relacionales, propios de las bases
de datos relacionales.

Vemos dos tecnologías.

- Dapper. Más sencillo. Para introducirnos.

- Entity Framework.

# Controlador

Controlador. Permiten conectar bases de datos relacionales con lenguajes de
programación orientados a objetos. Provee una capa de abstracción sobre los
diferentes proveedores de bases de datos.  Podemos escribir consultas `SQL`
directamente en `C#` o trabajar directamente en _orientación a objetos_ y que el
lenguaje se encargue de la prelación con la base de datos.

# NUGET

Para incorporar controladores a nuestro proyecto. Podemos usar un gestor de
paquetes como es `NUGET`. Para incorporar un paquete usamos:

```sh
dotnet package add <paquete>
```

`dotnet` lo incopora al proyecto, dentro del Archivo `.csproj`:

```xml
<ItemGroup> 
<PackageReference Include="Dapper" Version="2.1.79" />
</ItemGroup> 
```

Instala paquetes en `$HOME/.nuget`. En ese directorio se puede revisar el código
del paquete.

Ayuda con el gestor de paquetes:

```sh
dotnet package --help
```

`nuget.org`. Front-end web del repositorio. Ofrece información sobre los
paquetes.

# Dapper

Es una biblioteca ORM (Object-Relational Mapping, mapeador objeto-relación)
simple. Permite vincular bases de datos relacionales con un lenguaje de
programación orientado a objetos como es `C#`.

Instalamos Dapper:

```sh
dotnet package add Dapper
```

Esto no cambia el código. Pero cambia la definición de nuestro proyecto. Ahora
hace referencia a Dapper.

Incorporamos Dapper al proyecto:

```csharp
using Dapper;
```

Cadenas de conexión. Variable de tipo string. Por ejemplo, para `SQLite`:

```csharp
string connectionString = "Data Source=file.db";
```

Con esto indicamos a Dapper dónde buscar la base de datos.

Incorporamos la base de datos.

```csharp
using Microsoft.Data.SSqlite;
```

Recursos no administrados. Cuando tenemos recursos que no son propios de `.NET`
tenemos que manejar nosotros la memoria. Por ejemplo, si solicitamos información
a la base de datos, a través de una conexión, la gestión de los recursos de la
conexión dependen de nosotros.

Recursos propios. El _garbage collector_ se encarga. Recursos no administrados.
Nos tenemos que encargar nosotros.

Bloque `using`:

```csharp
using (var connection = SqliteConnection)
{
}
```

`connection` va a ser local al bloque `using`.  De esta forma, el _framework_ se
encarga de cerrar la conexión de forma segura.
También _excepciones no controladas_.

Dentro del bloque:

```csharp
using (var connection = SqliteConnection)
{
    // Abrimos la conexión
    connection.Open();
    // Creamos la sentencia SQL con string de más de una línea @""
    string createTable = @"CREATE TABLE IF NOT EXIST Peliculas(
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      titulo TEXT NOT NULL,
      genero TEXT NOT NULL
    )";

    // Ejecutamos la consulta
    connection.Execute(createTable);

    // Consulta para insertar datos
    string insertDune = "INSERT INTO Peliculas(titulo, genero) VALUES('Dune', 'Ciencia Ficcion')";

    // Ejecutamos la consulta INSERT
    connection.Execute(insertDune);
}
```

Hasta aquí usamos consultas mendiante una conexión directa a `SQLite`. Ahora
veremos cómo usar el `ORM`.

# Clase POCO

Creamos la clase. Como está en `SQLite`.

```csharp
public class Pelicula
{
    public string Titulo { get; set; }
    public string Genero { get; set; }
}
```

Una tabla no tiene métodos ni campos privados. Usa directamente _Propiedades_ de
C#. Los tipos de datos deben ser _primitivos_.

Un objeto compuesto por _propiedades primitivas_ se denomina `POCO`: 
_Plain Old C# Object_.

# Consulta usando Dapper

```csharp
using (var connection = SqliteConnection)
{
    // ...

    // Queremos hacer un SELECT, que devolverá registros
    // Por ello usamos el método QueryFirst<clase>
    string consulta = "SELECT * FROM Peliculas WHERE titulo = 'Dune'";

    // Lo guardamos en la clase película
    Pelicula pelicula = connection.QueryFirst<Pelicula>(consulta);
    
    // Imprimimos el resultado
    Console.WriteLine($"Título: {pelicula.Titulo}, género: {pelicula.Genero}")
}
```
