# Controladores

Controladores. Permiten conectar bases de datos relacionales con lenguajes de
programación orientados a objetos. Se puede abstraer a distintos niveles.
Podemos escribir consultas `SQL` directamente en `C#` o trabajar directamente
en _orientación a objetos_ y que el lenguaje se encargue de la prelación con la
base de datos.

# NUGET

Repositorio de paquetes de `.NET`. Usamos:

`dotnet package add <paquete>`

Instala paquetes en `$HOME/.nuget`. Ahí podemos revisar el código del paquete.

`nuget.org`. Front-end web del repositorio. Podemos obtener información sobre
los paquetes ahí.

# Dapper

Es una biblioteca ORM (Object-Relational Mapping, mapeador objeto-relación)
simple. Permite vincular bases de datos relacionales con un lenguaje de
programación orientado a objetos como es `C#`.

Incorporamos Dapper:

`dotnet package add Dapper`

Esto no cambia el código. Pero cambia la definición de nuestro proyecto. Ahora
hace referencia a Dapper.

Cadenas de conexión. Variable de tipo string. Por ejemplo:

`string connectionString = "Data Source=file.db";`

Con esto indicamos a Dapper dónde buscar la base de datos.

Recursos no administrados. Cuando tenemos recursos que no son propios de `.NET`
tenemos que manejar nosotros la memoria. Por ejemplo, si solicitamos usar una
conexión a la base de datos, tenemos que recordar dar de baja esa conexión.

Sentencia `using`. Para eso usamos:

```csharp
using Microsoft.Data.SSqlite;

// ...

using (var connection = SqliteConnection)
{
}
```

De esta forma, `connection` va a ser local al bloque de código de la conexión.
Así el _framework_ se va a encargar de cerrar la conexión de forma segura.

Dentro del bloque:

```csharp
using (var connection = SqliteConnection)
{
    // Abrimos la conexión
    connection.Open();
    // Creamos la sentencia SQL con string de más de una línea @""
    string createTable = @"CREATE TABLE IF NOT EXIST Peliculas(
      id INTEGER PRIMARU JEY AUTOINCREMENT,
      titulo TEXT NOT NULL,
      genero TEXT NOT NULL
    )";

    // Ejecutamos la query
    connection.Execute(createTable);

    // Query para insertar datos
    string insertDune = "INSERT INTO Peliculas(titulo, genero) VALUES('Dune', 'Ciencia Ficcion')";

    // Ejecutamos INSERT
    connection.Execute(insertDune);
}
```

# Clase película

Creamos la clase. La modelamos como está en la base de `SQLite`.

```csharp
public class Pelicula
{
    public string Titulo { get; set; }
    public string Genero { get; set; }
}
```

Una tabla no tiene métodos ni campos privados. Por lo tanto, recurrimos
directamente a _Propiedades_ de C#. Este tipo de objeto se denomina `POCO`:
_Plain Old C# Object_.

# Consulta

Hacemos una consulta usando `Dapper`.

```csharp
using (var connection = SqliteConnection)
{
    // ...

    // Queremos hacer un SELECT, una consulta que retoma registros
    // Por ello usamos el método QueryFirst<clase>
    string consulta = "SELECT * FROM Peliculas WHERE titulo = 'Dune'";

    // Lo guardamos en nuestra recién creada clase Pelicula
    Pelicula pelicula = connection.QueryFirst<Pelicula>(consulta);
}
```
