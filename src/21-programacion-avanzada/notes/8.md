Vimos SQL Server y SQLite. Vimos cómo vincular C# y SQLite con Dapper. Hoy
trabajamos con `Entity Framework`.

Ya no vamos a escribir consultas SQL. Trabajamos en el paradigma orientado a
objetos y `Entity Framework` se encarga de las consultas.

Para eso vamos a modelar objetos usando clases `POCO`. Una clase que solo usa
propiedades públicas y que representa la tabla de `SQL`.

# Entity Framework

Creamos el proyecto. 

```sh
dotnet new console --use-program-main -o Peliculas
```

Probamos que funcione.

```sh
cd Peliculas
dotnet run # => "Hello, World!"
```

Creamos el archivo `Pelicula.cs` para definir la clase POCO `Pelicula`.

```csharp
namespace Peliculas;

// Por convención en C# las clases llevan el mismo nombre del archivo
public class Pelicula
{
    public int ID { get; set; }

    public string Titulo { get; set; }

    public string Genero { get; set; }
}
```

En el método `Main` tomamos _input_ del usuario. Guardamos esa información en
la clase POCO.

```csharp
static void Main(string[] args)
{
    int id = Convert.ToInt32(Console.ReadLine());
    string titulo = Console.ReadLine();
    string genero = Console.ReadLine();

    // TODO(dan): Comprobar si esto se puede asignar así
    Pelicula pelicula = new Pelicula({
        ID = id,
        Titulo = titulo,
        Genero = genero,
    });
}
```

Incorporamos `Entity Framework` al proyecto.

```sh
dotnet package add Microsoft.EntityFrameworkCore.Sqlite
```

Este comando descarga el paquete (si no lo teníamos) a `$HOME/.nuget` y modifica
el archivo `Peliculas.csproj`.

```xml
<ItemGroup>
  <PackageReference Include="Microsoft.EntityFrameworkCore.Sqlite" Version="10.0.8"/>
</ItemGroup>
```

Instalamos una herramienta para `dotnet`: `dotnet-ef`. Se instala de forma
global.

```sh
dotnet tool install --global dotnet-ef
```

Así ya podemos trabajar con `Entity Framework` en nuestra aplicación.

# Clase DBContext

Para usar Entity Framework creamos una clase que herede de `DbContext`. En el
archivo `ApplicationDbContext.cs`.

```csharp
using Microsoft.EntityFrameworkCore;

namespace Peliculas;

// La clase se llama igual que el archivo, hereda de DbContext, que viene del 
// archivo que mencionamos en la directiva using de arriba
public class ApplicationDbContext : DbContext
{
    // Propiedad que representa una tabla
    // La propiedad lleva el nombre de la tabla
    public DbSet<Pelicula> Peliculas { get; set; }
    // Si tuviéramos una tabla Directores en Sqlite
    //public DbSet<Director> Directores { get; set; }

    // Le indicamos cómo conectarse a Sqlite
    // sobreescribien el método OnConfiguring
    protected override void OnConfiguring(DbContextOptionsBuilder options)
    {
        // La única opción que necesitamos es especificar el archivo .db
        options.UseSqlite("Data source=file.db");
    }

}
```
