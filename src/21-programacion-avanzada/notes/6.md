# Base de datos

Se usa para la persistencia de la información.

Hasta ahora manejamos información en memoria. Termina el programa y se termina
la información.

Persistencia es que la información permanece luego del ciclo de ejecución de
nuestra aplicación.

Solo se almacena por requerimiento nuestro.

No puede haber efecto secundario de pérdida de información, por ejemplo, por un
corte del suministro de energía. La información solo puede ser eliminada cuando
explícitamente lo pedimos. Eso define a la persistencia.

# Tipos de bases de datos

Hay relacionales y no relacionales. Aquí vamos a trabajar exclusivamente con
relacionales.

Son relacionales porque la información se almacena en tablas.

Que un campo haga referencia a un registro de otra tabla no refiere a las
relaciones. Las tablas hacen referencia las relaciones (investigar).

# Ingresar, consultar, actualizar y eliminar

Se hace de forma explícita. No puede ser efecto secundario de otra acción
dentro de nuestro software.
