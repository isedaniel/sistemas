# Base de datos

Persistencia. Implica permanencia de la información luego del cierre de la
aplicación. Hasta ahora solo usamos la Memoria Ram volátil. La persistencia
implica utilizar el almacenamiento. Para ello vamos a recurrir a un _motor de
bases de datos_.

El motor permite insertar, leer, editar y remover (**CRUD**) información
_explícitamente_. Evitando efectos secundarios como corrupción de los datos por
un corte en el suministro eléctrico.

Bases de datos relacionales. En esta materia solo usaremos bases de datos
relacionales.

Relación. Refiere a la organización de la información en tablas. Tablas y
relación son dos formas de refererirse a lo mismo. Las tablas establecen
relaciones. Así como las filas son registros, las tablas son relaciones.

Consultas. Son solicitudes para insertar, leer, editar y remover información.
Para ello usamos el lenguaje `SQL`.

`SQL Server`. Es el motor de base de datos relacional que se utiliza
generalmente en el ecosistema `.NET`. Inspirado en el estándar `SQL`, pero con
sus propios detalles de implementación. Igual que el resto de implementaciones,
como Oracle o PostgreSQL. Tenemos que descargar e instalar `SQL Server` y
`SSMS`.

En la materia vamos a usar dos sistemas:
- SQL Server: primero
- SQLite: luego, para usar un motor minimalista.

Identificación en `SQL Server`. `<hostname>/<instancia>`. Tildamos certificado
servidor de confiaza. Esto es necesario porque la instancia tiene un
certificado autofirmado. No es una práctica recomendable con un servidor
externo, pero es aceptable si es una instancia propia. Más adelante vamos a
profundizar en temas de cifrado.

Tabla. Tiene que pertenecer a una base de datos. Por lo que luego de loguearnos
a la instancia, tenemos que crear la base y ahí incorporamos la tabla. Tenemos
asistentes gráficos para generar las tablas. (Aunque mejor manual para
practicar `SQL`).

# Creando la tabla

Identificación. Los registros pueden compartir ciertos campos. Como el nombre.
Por eso creamos uno distinto para cada uno: un campo `id`. Lo marcamos `AUTO
INCREMENT`, para que el motor maneje el número.

# Insertar información

`INSERT INTO Peliculas(Titulo, Genero, FechaDeEstreno)
VALUES('Superman', 'Superhéroes', '1/1/1960');`

Respuesta. Cuántas filas fueron afectadas. En este caso 1. Porque modificamos 1
registro.

# Consultar información

```sql
SELECT TOP(1000) PeliculasId, Titulo, Genero, FechaDeEstreno
FROM Peliculas;
```

Devuelve los campos `PeliculasID`, `Titulo`, `Genero`, `FechaDeEstreno`, de la
tabla `Peliculas`.

Filtro. Podemos filtrar con la sentencia `WHERE`. Se pueden usar operadores
lógicos como `OR` o `AND`. Se puede incorporar la sentencia `LIKE`, para ver
que el campo _contenga_ la información que le estamos solicitando.

```sql
SELECT * FROM Peliculas
WHERE Genero LIKE 'Super' AND FechaDeEstreno = '1/1/1960';
```

# Modificar información

Utilizamos la palabra `UPDATE`. Se pueden poner restricciones con `WHERE`.

```sql
UPDATE Peliculas SET Genero = 'Superhéroes' WHERE FechaDeEstreno LIKE '1960';
```

# Eliminar información

Utilizamos `DELETE`. Podemos filtrar con `WHERE`.

```sql
DELETE FROM Peliculas WHERE PeliculasID = 3;
```

# Relaciones 1:M

Relaciones uno a muchos. Un escritor puede escribir muchos libros. Por lo que
el libro está restringido: solo puede referir a un escritor.

Para saber qué tabla debe incorporar un campo para referir a otra tabla debemos
pensar en cuál de las tablas es la que está restringida.

Establecer una referencia. Agregamos el campo `DirectorID` a la tabla
`Peliculas`. Creamos la tabla `Directores`. Y seteamos las películas para que
apunten a los directores de la nueva tabla:

```sql
UPDATE Peliculas SET DirectorID = 2 WHERE PeliculaID = 3;
```

# SQLite

Base más sencilla. No tenemos una instancia en ejecución. Se utiliza embebido
en aplicaciones. Forma parte del proceso de nuestro programa.

Herramientas. Permite descargar herramientas de línea de comando.

`sqlite3`. Herramienta de línea de comando. Se puede ejecutar con un archivo de
base de datos como primer argumento: `sqlite3 file.db`. Es una herramienta para
administrar el archivo de base de datos que hemos seleccionado. Pero en nuestra
aplicación estará embebido.
